# Раздел «Закрытие смены» · бриф для Codex

> Кладётся в `docs/`. Читать вместе с `PROJECT.md`, `AGENTS.md`. Источник логики — лист «Данные» рабочей Google-таблицы Etna (журнал закрытия смен). Этот бриф — перенос ТОЛЬКО формы, её логики и записи данных. Усложнения старой системы намеренно НЕ переносятся (см. «Вне объёма»).

## Смысл

Сотрудник в конце смены вводит кассовые показатели за день. Система считает производные значения, пишет одну запись за дату и отправляет отчёт в Telegram. Это замена листа «Данные» — той же формы и той же логики, но на чистом backend вместо Google Sheets + Apps Script.

## Поля формы

Что вводит сотрудник — помечено **(ввод)**. Что считается само — **(авто)**.

Смена:
- Дата **(авто)** — текущая смена. Правило: смена, закрытая до 06:00, относится к предыдущему календарному дню (параметр «закрывается до» = 6).
- Сотрудник **(авто)** — из PIN-входа.

Касса на открытии:
- Откр. (расч.) **(авто)** — переносится из прошлой смены (см. «Логика», п. перенос).
- Откр. (факт) **(ввод)** — фактический остаток в кассе на начало.
- Откр. Δ **(авто)** = Откр. факт − Откр. расч.

Доходы:
- Терминал 1 **(ввод)**, Терминал 2 **(ввод)**, Яндекс.Еда **(ввод)**
- Безнал итого **(авто)** = Терминал 1 + Терминал 2 + Яндекс.Еда
- Наличные **(ввод)**, Переводы **(ввод)**
- Выручка итого **(авто)** = Безнал итого + Наличные + Переводы

Расходы из кассы:
- Мойка **(ввод)**
- Кальяны (шт) **(ввод)** → Выпл. кальяны **(авто)** = шт × ставка сотрудника-кальянщика (ставка на сотруднике, не глобальная — см. «Связи с модулями»)
- Доп. расходы **(ввод)**

Прочее (учитывается, НЕ вычитается из кассы):
- Такси **(ввод)** — логируется; в расчёт остатка кассы не входит. Параметр-предел на смену = 2000 (0 = выключено).

Инкассация:
- Инкассация **(ввод)** — сумма, изъятая из кассы.

Касса на закрытии:
- Остаток (факт) **(ввод)** — фактический пересчёт кассы на конец.
- Остаток (расч.) **(авто)** — см. «Логика».
- Разница **(авто)** = Остаток факт − Остаток расч. Подсветить, если |Разница| > предел (500).

План/факт:
- План выручки **(авто)** — по дню недели из настроек.
- % выполнения **(авто)** = Выручка итого / План.

Опционально (в старой форме не было, но полезно): свободный комментарий к смене.

## Логика (проверено на реальных строках)

- Безнал итого = Терминал 1 + Терминал 2 + Яндекс.Еда
- Выручка итого = Безнал итого + Наличные + Переводы
- Выпл. кальяны = Кальяны (шт) × ставка сотрудника с обязанностью «Кальянщик»
- **Остаток (расч.) = Откр. (факт) + Наличные − (Мойка + Выпл. кальяны + Доп. расходы) − Инкассация**
  - Такси в этой формуле НЕ участвует (подтверждено сверкой строк).
- Разница = Остаток (факт) − Остаток (расч.)
- % выполнения = Выручка итого ÷ План выручки
- Перенос: Откр. (расч.) текущей смены = Остаток (факт) предыдущей смены (физически в кассе остаётся то, что реально пересчитали).

## Решения по умолчанию (исторические данные были неровные из-за ручных правок — в новой версии берём один чистый вариант; подтвердить у владельца)

1. Безнал итого считается из трёх полей (Терм.1 + Терм.2 + Я.Еда). Если есть ещё источники безнала — добавлять отдельными полями, а не ручным переопределением итога.
2. Перенос остатка — от Остаток (факт) прошлой смены (см. выше). В старой таблице местами тянулся то «факт», то «расч.» — это чинится единым правилом.
3. Подсветка расхождения кассы при |Разница| > 500, но закрытие смены не блокируется.
4. Все денежные поля по умолчанию 0, если не заполнены.

## Запись данных

Одна запись на дату в таблице `shift_closings` (новый Postgres). Поля = колонки формы выше (ввод + авто). Плюс: `created_at`, `submitted_by`. Это прямой аналог одной строки листа «Данные».

Историю из листа «Данные» можно один раз импортировать в `shift_closings` (отдельная задача, на работу новой формы не влияет).

## Telegram

При закрытии смены backend отправляет отчёт в Telegram (бот — как раньше). Адресаты из настроек: «руководитель» (закрытый чат) и «команда» (общий чат). Содержимое отчёта: дата/день/сотрудник, разбивка доходов, расходы, инкассация, касса (факт/расч/разница с пометкой при превышении предела), план и %.

ВАЖНО: точный текст/формат текущего сообщения в этот бриф не попал. Если нужен формат 1:1 — приложить пример реального сообщения, тогда воспроизвести дословно. Иначе собрать чистый отчёт по полям выше.

## Настройки, которые нужны форме (подмножество старых)

- Предел расхождения кассы = 500
- Смена закрывается до (час) = 6
- Предел такси на смену = 2000
- План выручки — НЕ хранится здесь. Берётся из модуля «График» (там считается под правами руководителя) — единый источник плана.
- Ставка за кальян — НЕ глобальная. Задаётся на сотруднике с обязанностью «Кальянщик».
- Сотрудники + PIN — для входа (значения только в защищённом хранилище, не в коде).
- Telegram: токен бота и chat_id — только через `.env`.

## Связи с другими модулями

- **План ← «График».** График уже рассчитывает план выручки на дату под правами руководителя; закрытие смены берёт готовое значение, своей таблицы планов не держит. Если график отдаёт диапазон — для % выполнения берётся **верхняя граница** диапазона.
- **Кальяны → выплаты и ЛК.** Поле «Кальяны (шт)» относится к сотруднику с доп. обязанностью «Кальянщик»: количество и заработок (шт × ставка) идут в его выплаты и в статистику его ЛК. Ставка задаётся на сотруднике (там же, где роль/часы/ставка), не в настройках формы. В смене один кальянщик — поле «Кальяны» одно и вешается на этого сотрудника. «Выпл. кальяны» — это и расход кассы при закрытии, и заработок сотрудника: одни деньги в двух разрезах, не дублировать как два разных списания.

## Вне объёма (НЕ переносим)

- Закуп: фудкост 25%, дневной лимит, перенос/накопленный остаток.
- Кэш-флоу: прогноз на 45 дней, регулярные платежи, резерв закупа, сверка расчётного счёта.
- Бот расходов и категории cash-flow, аналитический дашборд.
- Дашборд можно собрать заново в новом сервисе ПОЗЖЕ, поверх `shift_closings` — но это отдельная задача, не часть переноса формы.

## Безопасность

- Telegram bot token из старой таблицы считать скомпрометированным — перевыпустить. PIN-ы сменить. В код/гит секреты не писать — только `.env` / защищённые настройки.
