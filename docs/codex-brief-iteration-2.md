# Бриф · Итерация 2 (UX + новый функционал)

Зачем: структурировать пакет правок и нового функционала от руководителя в правильный
порядок реализации. Источник — голосовой/текстовый список из 6 разделов. Этот документ —
рабочий план: читать перед стартом, вести по нему фазы. Статусы проставлять прямо здесь.

Снято: 2026-06-06. Контекст: кабинет уже в тёмной теме RestForm (антик-золото, Unbounded+Golos
Text), боевой адрес `https://lk.no-money-no-honey.ru/`, деплой — scp в `/opt/etna/app` + пересборка
образа `api` на Beget (`etna-vps`). Данные — Postgres через backend (`apps/api`), фронт — `apps/web`.

---

## 0. Как читать этот план

Каждый пункт помечен типом сложности:
- **[F]** — только фронт (`apps/web`), без БД. Быстро, низкий риск.
- **[FB]** — фронт + backend-эндпоинт, без новой схемы.
- **[FBM]** — фронт + backend + **миграция схемы** (`migrations/`, `migrate()`/`schemaVersion`).
  Самые дорогие, делать осознанно, обновлять `data-model.md`.

Инвариант (из AGENTS.md): рендер/расчёты ходят в backend, не в localStorage; при правке схемы —
обновлять миграции и `schemaVersion`; деньги/часы — разные типизированные сущности; цвета только
через CSS-переменные (`--brand`/`--brand-bright`/семантику), без захардкоженных hex.

## 2. Детализация по разделам

### Раздел 1 — Главная и ЛК
- **1.1 [F] Синий текст названий разделов — ДИАГНОЗ ПОДТВЕРЖДЁН.** Проявляется только на телефоне (iOS
  Safari), на десктопе нормально. Причина: `.navcard` — это `<button>`, а у кнопок в iOS системный
  цвет текста по умолчанию синий, если `color` не задан явно. В `styles.css` у `.navcard` нет `color`.
  **Фикс (1 строка):** добавить `color:var(--ink)` в правило `.navcard`. (Заодно проверить остальные
  кликабельные `<button>` без явного `color` — задать `color:inherit/var(--ink)`, чтобы не всплыло
  на других экранах телефона.) Это НЕ про `serviceMeta`-акценты — те (золото/роли) корректны.
- **1.2 [F] «Etna» → «ЭТНА».** `renderHub`: `<h1 class="brand-title">Etna</h1>` → `ЭТНА`. Проверить
  логотип-плитку «E» на PIN (там буква E) — возможно тоже «Э».
- **1.3 [F] Новый логотип — АССЕТ ПОЛУЧЕН ✅.** Файл сохранён в репо: `apps/web/assets/logo.png`
  (PNG 1039×1039, RGBA). Содержимое: золотая скруглённая плитка (≈`#cdb079`) со стилизованной чёрной
  «ЭТНА» и этно-орнаментом; прозрачны только углы. По сути готовая иконка-плитка в нашей гамме.
  Интеграция (Фаза A):
  - **Экран PIN:** заменить текущий `.logo` (золотой квадрат + `<span>E</span>`) на `<img src="/assets/logo.png">`
    того же размера (74×74, border-radius уже у плитки свой — можно убрать фон/тень у `.logo` контейнера).
  - **Шапка ЛX:** т.к. плитка уже содержит «ЭТНА», варианты: (а) маленькая плитка-иконка вместо `.mark`-бара +
    убрать текстовый `brand-title`; (б) оставить текст «ЭТНА» (п.1.2), плитку использовать только на PIN.
    Решить на старте — рекомендую (а) для единства бренда.
  - Цвет золота в логотипе чуть теплее `--brand` — допустимо, не подгонять.
- **1.4 [F] Убрать дату в углу.** `renderHub`: удалить `<span class="top-meta">${today}</span>`
  (дата уйдёт в приветствие — см. 1.5).
- **1.5 [FB] Приветствие с целью дня.** Текст: «Привет, {Имя}. Сегодня {день недели}, {дата}, и наша
  цель сегодня:» + два показателя:
