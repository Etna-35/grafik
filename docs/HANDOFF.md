## 2. Стек и структура

## 3. Визуальный стиль (бренд RestForm) — строго соблюдать
Тёмная тема, перенесена из соседнего проекта `Etna-35/RestForm` (источник правды по бренду).

  светлое `--brand-bright:#e2c98a`, **текст на золоте тёмный `--brand-ink:#0e0e0e`** (не белый!).
  Семантика: `--green/--gold/--danger` + `--ok-bg/--ok-line`, `--danger-bg/--danger-line`, `--brand-tint/--brand-line`.
- Шрифты: дисплейный **Unbounded** (`--font-display`: заголовки, золотые метки, суммы, клавиши),
  текст **Golos Text** (`--font`). Подключены в `apps/web/index.html`.
- Mobile-first. Метки минималистичные монохромные, без emoji в UI.

## 4. ПРОЦЕДУРА ДЕПЛОЯ (важно — повторять ровно так)
Изменения едут на сервер копированием файлов + пересборкой образа `api` (фронт запечён в образ!).

1. Правки в коде. Проверка: `npm run check` (tsc) и `node --check apps/web/app.js`.
2. (опц.) Визуальная проверка: Claude Preview / launch.json `web-static` (python http.server по `apps/web`).
   ВНИМАНИЕ: браузер превью кэширует `styles.css` — после правок CSS делать reload и/или cache-bust
   (`link.href='/styles.css?b='+Date.now()`), иначе увидишь старые стили.
3. Коммит в `main`, пуш: `GIT_TERMINAL_PROMPT=0 git push origin main`.
   Если пуш падает с HTTP 400/RPC (большие бинарники): уже выставлен `git config http.postBuffer 524288000`.
4. Залить файлы на сервер (примеры):
   - `scp apps/web/app.js apps/web/styles.css etna-vps:/opt/etna/app/apps/web/`
   - `scp apps/api/src/<файлы>.ts etna-vps:/opt/etna/app/apps/api/src/`
   - новые подпапки: сначала `ssh etna-vps "mkdir -p /opt/etna/app/apps/api/src/data"`
   - миграции: `scp migrations/NNN.sql etna-vps:/opt/etna/app/migrations/`
   - ассеты: `scp apps/web/assets/awards/*.png etna-vps:/opt/etna/app/apps/web/assets/awards/`
5. Пересборка и рестарт:
   `ssh etna-vps "cd /opt/etna/deploy && docker compose build api && docker compose up -d api"`
6. Миграции применяются АВТОМАТИЧЕСКИ при старте контейнера (entrypoint → `node apps/api/dist/migrate.js`),
   затем seed. Проверить лог: `docker compose logs api | grep -i 'Applied migration'`.
7. Импортёры запускать вручную в контейнере после деплоя:
   `ssh etna-vps "cd /opt/etna/deploy && docker compose exec -T api node apps/api/dist/importMenu.js"`
8. Проверка: `curl -s -o /dev/null -w '%{http_code}' https://api.no-money-no-honey.ru/api/health` → 200.
   БД-проверки: `docker compose exec -T postgres psql -U etna_app -d etna_app -tAc "..."`.

Миграции: только вперёд, нумерация `NNN_name.sql` (последняя — `021_quiz.sql`). Идемпотентность желательна
(`IF NOT EXISTS`, `DROP CONSTRAINT IF EXISTS`). При правке схемы — обновлять `docs/data-model.md`.

В git-коммитах подпись: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.

## 5. Что уже построено (готово, на проде)
- **ЛК/главная:** приветствие + «цель дня» (план выручки = ФОТ_дня/0.23 ×0.8 будни/×1.5 ПТ-СБ; наличный
  план = ФОТ дня), единая подложка с блоками (План выручки / Наличный план / План от коллег).
  Зона **«Заслуги»**: блок Уровня (клик → страница прогресса) + «Задачи на месяц N/N»; ниже похвалы/оценки/«Спасибо».
- **График** (`schedule`): смены, роль-скоуп, оценки (зелёный/жёлтый/красный), выплаты, ДР-метки в календаре, ФОТ.
- **Закрытие смены** (`shiftClosing`): полная форма; переключатель даты ←/→, время МСК + ночное предупреждение
  (00:00–06:00). Отменены пользователем: такси «за наличные», 2 кальянщика (НЕ реализовывать без запроса).
- **Задачи** (`tasks`): описание, дедлайн, **премия** (→ в Выплаты); статус у названия; ролевые задания
  («вся смена» — видно только роли, отметка о выполнении, +10% всем роли). «План на завтра» (`handovers`).
- **Заявка** (`requisitions`): каталог, срочность по позиции, золотые кнопки.
- **Выплаты** (`payroll`): начисления по сменам, кальяны, премии за задачи и за цели продаж.
- **Админка** (`admin`): сотрудники (PIN/роль/ставки/доступы), дата начала + ДР, **удаление (архивация)** с подтверждением.
- **Похвала коллеге** (`praises`) + лента.
- **Цели по продажам** (`salesGoals`): руководитель ставит цель официанту (позиция/кол-во/премия),
  сотрудник жмёт «Продал +1», при достижении руководитель подтверждает → премия в доход. Блок на главной.
- **Прогресс/геймификация** (`progress`): уровень = 100 очков; стаж +30%/мес, задание рук-ля +30,
  ролевое +10 (всем роли), похвала коллеги +10 / рук-ля +20, кнопка «План выполнен» +50 всем.
  Страница прогресса (история «как в MK», скрытый следующий уровень). Иконки уровней — **20 кукол-вуду**
  (`apps/web/assets/awards/level-1..20.png`, `MAX_MASK_LEVEL=20` в app.js).
- **Обучение + Тесты** (`training`, `quiz`): 3 независимых блока (база официанта, меню кухни, меню бара).
  Глава → тест (таймер = вопросов×90с, без подсказок, порог 80%, провал → блок на 2 ч, успех → следующая глава);
  аттестация по модулю. Меню кухни/бара импортированы из ТТК (87 вопросов глав + 26 аттестаций).
