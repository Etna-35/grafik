# Бриф · Движок обучения с тестами (Quiz)

Зачем: реализовать тестирование внутри раздела «Обучение» по требованиям руководителя.
Автономная задача (исполняется без уточнений). Этот документ — план и статус.

## Требования (от руководителя)
- Сотрудник читает главу. Чтобы перейти дальше — пройти тест по главе.
- Кнопка «Начать» → общий таймер = (кол-во вопросов × 90 сек).
- Во время теста НЕ показываем правильность ответов.
- В конце — результат в % без разбора, где ошибся.
- < 80% → сообщение «доизучи и вернись через 2 часа» (ретест заблокирован на 2 ч).
- Прошёл (≥80%) → открывается следующая глава и её тест (тот же принцип).
- В конце раздела — общий тест «Аттестация» по всему разделу.
- Отдельные блоки знаний: меню кухни и барное меню — изучаются по блокам с тестами так же.
- После — общая аттестация отдельно по кухне и отдельно по бару.

## Источники контента (доступны)
- ТТК Google Sheets (CSV-экспорт по gid работает) и xlsx на Google Drive (`1KID9...`) —
  11 листов: кухня (Закуски, Салаты, Супы, Пицца/бургеры, Основные, Десерты) и
  бар (Коктейли Классика/Авторские, Моктейли/лимонады, Авторские чаи, Настойки).
- Гайд по предметам интерьера (Google Drive PDF `1r4Dk...`) — для будущей главы «Интерьер».

## Модель данных (миграция 021)
- `quiz_questions(id, chapter_id?, module_id?, is_attestation, prompt, sort_order, is_active)`
  — вопрос относится либо к главе (тест главы), либо к модулю с is_attestation (аттестация).
- `quiz_options(id, question_id, label, is_correct, sort_order)`.
- `quiz_attempts(id, employee_id, scope_type[chapter|attestation], scope_id, total, correct,
  score_pct, passed, started_at, finished_at)`.

## Правила (константы)
- Порог = **80%**. Время на вопрос = **90 сек**. Блокировка после провала = **2 часа**.

## Гейтинг
- Глава доступна, если тест предыдущей главы пройден (первая — всегда).
- Аттестация модуля доступна, когда пройдены тесты всех глав модуля.
- Ретест: если последняя попытка не пройдена и прошло < 2 ч — заблокировано до `finished_at + 2ч`.

## API (`quiz.ts`)
- `GET /api/training/quiz/:scope/:id` → состояние (canStart, passed, lockedUntil, questionCount, durationSec).
- `POST /api/training/quiz/:scope/:id/start` → создаёт попытку, отдаёт вопросы БЕЗ флага верного ответа.
- `POST /api/training/quiz/attempts/:attemptId/submit` → серверный подсчёт, % и passed, без разбора.
- В `training init` к каждой главе добавлено: `quiz{questionCount,passed,lockedUntil,canStart}` + `locked`;
  к модулю — `attestation{...,available}`.

## Контент-модули
- Официант (существующий) — тесты на главы + аттестация (вопросы засеяны примером, дополняются руководителем).
- **Меню кухни** и **Меню бара** — новые модули; главы = листы ТТК; импортёр `importMenu`
  читает `apps/api/src/data/menu-content.json` (выгружен из xlsx). Автоген простых вопросов
  «Состав блюда X?» (верный = его состав, дистракторы = составы других блюд) + аттестация по модулю.
