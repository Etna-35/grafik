# План по блокам (согласовано с руководителем)

Обновлено: 2026-06-11. Это «продуктовый» бэклог по договорённости с владельцем — отдельно от

Порядок реализации: **Блок 1 → Блок 2 → Блок 4 → Блок 3** (сначала данные, в конце — сводная финмодель).

## Блок 1. Обучение и тесты — ✅ ВЫПОЛНЕН (коммит 6e8b294)
- 1.1 Дашборд руководителя: показывать всех сотрудников любой роли, у кого есть прогресс
  (сдан ≥1 тест); нулевых/неначавших скрывать; показывать число сданных тестов. ✅
- 1.2 Перерыв на пересдачу теста: 2 часа → **10 минут** (`QUIZ_LOCK_MINUTES=10`). ✅
- 1.3 Добровольный «тест-челлендж»: рандом из всей базы (главы+аттестации), **лимит 2/сутки**,
  **+20%** за пройденный. Без анти-повторов и без «конца контента» (по решению владельца).
  `scope_type='challenge'`, sentinel `scope_id` = нули; `CHALLENGE_LIMIT=15`, `CHALLENGE_PER_DAY=2`. ✅

## Блок 2. Выплаты (ЛК сотрудника) — 🔜 СЛЕДУЮЩИЙ
- 2.1 Выплата **с указанием месяца** — вычитать из указанного месяца, не из текущего по умолчанию.
- 2.2 Возможность привязать выплату к выбранному «Личному обязательству» (`employee_obligations`).
      Делать только если не усложняет расчёты/код.
- 2.3 Внести ЗП прошлых месяцев — **только данные учёта**, без заполнения календаря за прошлое.
      Владелец пришлёт цифры отдельно.
- 2.4 «История выплат» → **сворачиваемый** список (не всегда раскрыт). Видно: сколько/когда выдал,
      за какой месяц или за какое личное обязательство.
- 2.5 Перекомпоновка блоков ЛК «Выплаты»: «Начислено + Выдано» в один блок, «Смены + Часы» в один.
      «Остаток к выплате» → переименовать в **«Твой доход за текущий месяц»**.
      Под ним — общий **долг за прошлые месяцы** показывать только если есть невыданные деньги.
- 2.6 Убрать раздел **«Начисления по сменам»** (у сотрудника и у руководителя) — лишний.

## Блок 4. Заявки — после Блока 2
- 4.1 ~~Переименовать в «Заявка/списания»~~ — **ОТМЕНЕНО** (списания через сервис не ведём).
      Для поваров/барменов переключатели «Заявка» (вместо «Каталог») / «Мои заявки».
- 4.2 Матрица цен + единиц: владелец заполняет `docs/catalog-products.csv` и `docs/catalog-household.csv`
      (учётная фасовка «бутылка 800мл» vs единица заказа «шт»; цена за единицу). Я строю каркас импорта.
- 4.3 Контроль фудкоста по заявке: при заявке на крупную сумму — флаг «вероятен перезакуп»
      (зависит от 4.2 и норм из Блока 3).
- 4.4 Фактический закуп руководителем: «надо 5 кг, купил 3» — корректировка факта при отметке закупки.

## Блок 3. Финансы руководителя (P&L) — ПОСЛЕДНИМ, по частям
- 3.1 Форма постоянных платежей (аренда, связь, ПО, бухгалтерия…). Заполняет владелец.
- 3.2 ~~Telegram-бот учёта расходов~~ → **отдельная минималистичная страница** (значок на телефон):
      ввод суммы + тап-кнопки категорий; при заполнении строки появляется ещё одна + «отправить»;
      после отправки экран очищается и показывает подтверждение + «внести ещё». (Проще и безопаснее бота.)
- 3.3 Таблица P&L с нормами % от выручки: Food Cost 28%, Бар-кост 15%, Итого кост 43%, Хоз 1,8%,
      ФОТ 22% (факт из календаря), Аренда 8%, Коммуналка 3%, Маркетинг 2%, Бухгалтерия 1%,
      Интернет/связь/ПО 0,5%, Амортизация/ремонт 1%, Прочие 1,2%, Итого 85%, EBITDA 14,5%.
      Источники: закупки (заявки/страница расходов), ФОТ (календарь), постоянные (3.1).
- 3.4 Прогноз выручки месяца: средние по дням недели × отношение к текущему месяцу + фактические
      прошедшие дни. Для расчёта норм в реальном времени.

### Риск/порядок
Блок 3 связывает заявки (закуп), календарь (ФОТ) и расходы. Поэтому строго «снизу вверх»:
сначала данные (Блоки 2, 4 + страница расходов), потом сводная таблица P&L.
