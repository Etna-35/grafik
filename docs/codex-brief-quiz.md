# Бриф Codex · Квиз / Аттестация

## Задача

Сделать отдельный сервис `Квиз / Аттестация` на поддомене `https://quiz.no-money-no-honey.ru/`.

Квиз нужен, чтобы сотрудники в игровой форме закрепляли материалы из раздела `Обучение / База знаний`, а руководитель видел реальную картину прохождения и понимания материала.

## Архитектурное решение

- Квиз живет на отдельном поддомене: `https://quiz.no-money-no-honey.ru/`.
- Backend тот же, что у ЛК.
- БД та же: Postgres проекта Etna.
- Аккаунты те же: сотрудники входят теми же PIN-аккаунтами.
- Не делать отдельную авторизацию, отдельный список сотрудников или отдельную БД, иначе аттестация не свяжется с сотрудником.

## Связь с обучением

- Вопросы квиза могут быть привязаны к главам или модулям из `Обучение / База знаний`.
- Результаты квиза показываются сотруднику в ЛК как личный мотиватор.
- Результаты квиза показываются руководителю в дашборде: кто сколько прошел, активность, лучший результат, что осталось.
- Раздел обучения описан отдельно: `docs/codex-brief-training.md`.

## Решения по умолчанию

- Порог прохождения темы: `80%`.
- Общая аттестация: `пройдено / всего`.
- Пересдачи разрешены.
- В зачет идет лучший результат сотрудника по теме.
- Лидерборд выключен по умолчанию из-за приватности.
- Командное сравнение можно добавить позже только отдельным решением.

## Данные

Предлагаемые сущности:

- `quiz_topics` — тема аттестации, может быть связана с модулем или главой обучения.
- `quiz_questions` — вопросы темы.
- `quiz_answers` — варианты ответов.
- `quiz_attempts` — попытки сотрудника.
- `quiz_attempt_answers` — ответы сотрудника в конкретной попытке.

Вопросы и ответы должны редактироваться без кода через NocoDB/админку. Схему БД создает backend-миграция, не NocoDB.

## Права

- Сотрудник видит только свои попытки и свои результаты.
- Руководитель/менеджер видит результаты всех сотрудников.
- Доступ к квизу должен быть связан с теми же правами/назначениями, что и обучение.
- Нельзя показывать сотруднику чужие результаты, попытки или сравнения.

## Интерфейс сотрудника

На `quiz.no-money-no-honey.ru`:

- вход через существующую PIN-сессию или PIN-экран;
- список доступных тем;
- статус темы: не начато, в процессе, пройдено, не пройдено;
- прохождение вопросов;
- итог после попытки: процент, пройдено/не пройдено, лучший результат;
- подсказка, какие материалы повторить, если тема связана с главой обучения.

В ЛК сотрудника:

- блок результатов аттестации;
- сколько тем пройдено из общего количества;
- лучший процент по темам;
- что осталось пройти.

## Интерфейс руководителя

- Дашборд аттестации.
- Список сотрудников.
- Фильтр по роли, теме, статусу.
- Видно: попытки, лучший результат, дата последней активности, пройдено/всего.
- Без публичного лидерборда в MVP.

## API

Минимальный набор:

- `GET /api/quiz/init`
- `GET /api/quiz/topics`
- `GET /api/quiz/topics/{id}`
- `POST /api/quiz/topics/{id}/attempts`
- `GET /api/quiz/results`
- `GET /api/quiz/dashboard`

## Не делать в MVP

- Отдельный домен с отдельным backend.
- Отдельную авторизацию.
- Отдельную БД.
- Публичный лидерборд.
- Сертификаты и тяжелую LMS-логику.
- Интеграции с внешними LMS.

## Критерии готовности MVP

- `https://quiz.no-money-no-honey.ru/` открывает квиз-сервис.
- Квиз использует тот же backend, ту же БД и те же PIN-аккаунты.
- Сотрудник проходит тему, результат сохраняется в Postgres и привязан к его employee/user.
- Лучший результат используется как итоговый.
- Сотрудник видит свои результаты в ЛК.
- Руководитель видит дашборд аттестации.
- Рядовой сотрудник не видит чужие результаты.
