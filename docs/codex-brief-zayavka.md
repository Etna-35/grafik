# Раздел «Заявка» (продукты + хозтовары) · бриф для Codex

> Кладётся в `docs/`. Читать вместе с `PROJECT.md` (общая модель/фазы), `AGENTS.md` (правила) и `etna-zayavka-mockup.html` (визуальный референс именно этого экрана — зелёная подсветка выбранного).

## Что это и зачем

Экран, на котором сотрудник формирует **заявку на закупку** — список «что нужно купить», по двум типам: **продукты** и **хозтовары**. Заявка уходит управляющему / в закуп.

**Граница (важно, не путать с бюджетом):** заявка — это список потребностей, она НЕ списывает бюджет и НЕ является закупом. Фудкост и лимит 25% считаются отдельно, при закрытии смены (см. `PROJECT.md`). В UI это явно проговорено плашкой — сохранить этот смысл.

## База каталога (факты)

- Продуктовых позиций: **212**, категорий: **12**.
- Крупнейшие категории: Овощи 46, Соусы/бакалея/масла 28, Молочные/сыры/яйца 27, Алкоголь 22, Зелень 20. Остальные 7 категорий (~69 позиций) — в свёрнутом «ещё 7».
- Хозтовары — **отдельный набор** категорий и позиций (данных пока нет, в мокапе помечено «пример»). Тот же механизм поиска/добавления.
- Каталог сидируется из экспорта товарной базы Etna (см. «действия пользователя»).

## Поведение (по мокапу)

Две вкладки: **Каталог** и **Моя заявка**.

Каталог:
- Сегмент-тумблер **Продукты / Хозтовары** — переключает набор каталога (`kind`).
- **Поиск** по названию по всем позициям набора.
- Лента **категорий с живыми счётчиками**; «Все» — весь набор; длинный хвост свёрнут. У категории — цветной маркер.
- Строка позиции: название + единица измерения из базы. Не в заявке → кнопка «+» (зелёная). В заявке → счётчик «− кол-во ед +», строка подсвечена зелёным.
- Единицу измерения сотрудник НЕ меняет (берётся из каталога) — меняет только количество.
- Снизу липкая кнопка «В заявке N позиций · Оформить» → вкладка «Моя заявка».

Моя заявка:
- Автор (сотрудник) + куда уходит; счётчики: всего / продукты / хозтовары.
- Позиции, сгруппированные Продукты / Хозтовары, с количеством и единицей, удаление позиции.
- Поле комментария.
- Кнопка «Отправить заявку» (единственный брендовый-бордовый CTA; всё «выбранное» — зелёное).

## Модель данных (расширяет раздел 6 PROJECT.md)

Каталог (общий справочник):
- `categories`: `id, name, kind(product|household), sort, color?`
- `catalog_items`: `id, category_id, name, unit, kind(product|household), active`
  - `unit` — единица из базы (кг, шт, л, пучок, упак…). Снапшотится в строку заявки.

Заявка:
- `requisitions`: `id, author_id, created_at, status, comment, urgent(bool)`
- `requisition_lines`: `id, requisition_id, catalog_item_id, qty, unit(снапшот)`

Связи: `author_id` → сотрудник/пользователь (модель пользователей из фазы ЛК). Каталог — новые таблицы.

## Решения по умолчанию (если пользователь не уточнил — брать эти, спросить при сомнении)

1. **Статусы заявки:** `новая → принята → закуплена`, плюс `отклонена`. Менять статус может управляющий.
2. **Срочность:** флаг `urgent` на уровне заявки (одна галочка), не на позицию.
3. **Остаток/последняя цена у позиции:** в MVP сотруднику НЕ показывать. Возможно позже в виде управляющего.
4. **Свободная позиция «другое»:** разрешить добавлять строку свободным текстом (название + кол-во + ед.), помеченную как вне каталога, чтобы сотрудник не застревал, если позиции нет в базе.
5. **Роль-скоуп:** сотрудник видит и создаёт только свои заявки; управляющий видит все и меняет статусы. Проверка прав — в backend (инвариант из AGENTS.md).

## Объём первой реализации

- Модель каталога + сидирование из файла, который даст пользователь.
- Создание/отправка/список заявок, роль-скоуп.
- Два экрана строго по `etna-zayavka-mockup.html`.
- Можно делать UI-first на заглушечных данных (как в мокапе), затем подключить к API, когда поднимется БД из инфра-брифа.

Вне объёма сейчас: поставщики, цены/прайс, автоматическое превращение заявки в закуп/бюджет.

## Стиль (из мокапа, не отступать)

- Выбранное/добавление — зелёный токен `--ok` (`#2f7a52`): подсветка строки, рамка и кнопки счётчика, кнопка «+».
- Брендовый бордовый `#8f2433` — только главный CTA («Оформить»/«Отправить»).
- Остальное — общая система кабинета: Manrope + Playfair, бумажный фон, монохромные мелочи, mobile-first.
