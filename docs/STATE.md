# Текущее состояние · Etna · no-money-no-honey

Зачем: единый снимок «что реально построено и где». Читать вместе с `DECISIONS.md`. Цель — чтобы при сжатии контекста, новой сессии или передаче другому ИИ-агенту актуальное состояние находилось здесь, а не реконструировалось из кода. Обновлять при каждом значимом изменении.

Снято: 2026-06-04, по живому осмотру под правами руководителя.

## Деплой
- Beget VPS, Docker Compose: Postgres, Redis, NocoDB, Caddy, backend, фронт ЛК.
- Вход по PIN. Источник правды по данным — БД Postgres (НЕ файлы-снимки в `docs/`).

## Построено и проверено живьём
- **ЛК (хаб):** приветствие, бейдж роли, счётчики (Смен / Задач / Выдано), плитки сервисов.
- **График** (`/grafik`): на Postgres; роль-скоуп корректный — сотрудник видит общий календарь со сменами всех, но деньги/оценки только свои («Мой расчёт: Начислено / Выдано / Остаток»). Оценки — цветные точки; выплаты ведутся на календаре.
- **Закрытие смены** (`/smena`): полная форма по брифу. Доходы: Терминал 1, Терминал 2, **Нетмонет**, Яндекс.Еда → Безнал итого (авто); Наличные, Переводы → Выручка итого (авто). Касса открытия (расч./факт/Δ авто). Расходы: Мойка, Кальяны, Доп. расходы (списком). Такси — не из кассы, предел 2000 ₽. Инкассация. Касса закрытия: Остаток факт/расч (авто), «Разница» с порогом 500 ₽ (зелёная плашка в норме). Доступ — по индивидуальному праву.
- **Заявка** (`/zayavka`): каталог (актуальный, отредактирован руководителем, ~164 позиции, категории с цветными точками и счётчиками), табы Каталог/Моя заявка, Продукты/Хозтовары, поиск, зелёные «+». Плашка «не списывает бюджет и не влияет на фудкост». Доступ — по праву (обычно повара/бармены).
- **Админка** (`/admin`): сотрудники (14) с PIN/ролью/оплатой/часами/ставкой; **доп. обязанность «Кальянщик» (чекбокс) + индивидуальная ставка ₽/шт**; **доступы по каждому разделу (Видит / Меняет) на сотрудника**.
- **Задачи**: раздел есть, недавно доделан — НЕ протестирован.
- **Выплаты**: раздел «Начисления» есть.
- **Обучение** (`/training`): реализуется как база знаний официанта на Postgres. Источник стартового контента — Google Doc `Библия`, импортируется напрямую в БД через импортёр без сохранения текста в публичном git. Структура: 1 модуль, 7 глав, 5 дней маршрута стажёра, отметки `прочитано`, вложения-плейсхолдеры для артефактов/карты/ТТК, руководительский прогресс по сотрудникам.

## Источник правды по данным
- Боевая БД Postgres (через backend; админ-правки — через NocoDB/Админку).
- Каталог заявки: актуален в БД. `docs/etna-tovarnaya-matrica.xlsx` — ранний снимок, НЕ источник, пере-сидингом не пользоваться.

## Пробелы / проверить
- Категории заявки по ролям (бармену не «Мясо») — в админке виден доступ по разделам, фильтра по категориям внутри заявки не подтверждено.
- «Задачи» — нужен тест-проход.
- Код построчно (тесты, обработка ошибок, безопасность эндпоинтов) — не аудирован.
- Безопасность: репозиторий публичный (IP/адреса в README) → приватный; токен бота → перевыпуск.
