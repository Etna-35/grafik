## Зафиксированное решение

Для полноценного сервиса используем:

- `Postgres` как основную базу;
- небольшой backend как слой безопасности и бизнес-логики;
- `NocoDB` как админку руководителя;
- отдельный frontend личного кабинета.

`NocoDB` не должен быть публичным интерфейсом для сотрудников.

## Выбранная площадка для MVP

Для первого серверного запуска используем существующий VPS в РФ от Beget.

Купленная конфигурация MVP: `2 CPU 3-3.3 GHz / 4 GB RAM / 40 GB NVMe / 1 Gbit/s`, Ubuntu 24.04, IP `212.67.14.25`.

На VPS уже подняты `Postgres`, `Redis`, `NocoDB`, `Caddy`, backend API и frontend личного кабинета с PIN-входом.

Раздел `График` в ЛК уже умеет читать месяц из Postgres, импортировать JSON-бэкап текущего календаря и редактировать календарь через защищенный API для руководителя: смены, дедлайны, плановые дни зарплаты, фактические выплаты и точки оценки. Визуально ЛК ориентируется на старый график: компактная цветная сетка, колонка `Σ`, ФОТ/план и карточки итогов. Для обычного сотрудника график открывается в безопасном режиме: общий график и только свои точки оценок видны, а деньги, ФОТ, план выручки, ставки и месячные итоги не отображаются на странице графика. Внутренний раздел `/admin` в ЛК позволяет руководителю заводить сотрудников, менять PIN, параметры графика, ставку кальянщика и индивидуальные доступы к сервисам.

Раздел `Выплаты` в ЛК показывает личную финансовую информацию сотрудника отдельно от графика: начислено, выдано, остаток, смены, часы, ближайший плановый день выплаты и историю фактических выплат. Источник начислений — `schedule_shifts`, источник выплат — `payroll_payouts`.

Раздел `Задачи` показывает сотруднику только его задачи со статусами `В работе` и `Готово`. По команде сотруднику доступны только обезличенные числа: сколько задач всего и сколько выполнено другими. Руководитель/менеджер может назначать задачи сотрудникам в этом же разделе.

Раздел `Закрытие смены` в ЛК заменяет старую связку Google Sheets/Apps Script: форма пишет структурированные данные в Postgres, backend сам считает кассу, выручку, выплаты кальянщику, доп. расходы, отклонения и процент от плана. Поддерживаются поле `Нетмонет`, переводы в расчете остатка кассы, динамические доп. расходы, загрузка фото чеков и два Telegram-отчета: полный руководителю и короткий команде.
